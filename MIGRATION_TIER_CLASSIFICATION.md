## تعريف الـ Tiers

| Tier | التعريف | متطلبات النقل |
|------|---------|--------------|
| **Tier 1** — حرج جداً | يؤثر على بيانات العملاء، المال، الضرائب، الصلاحيات | Backup + Regression قبل + Dual Run + Approval قبل الحذف |
| **Tier 2** — مهم | يؤثر على العمليات اليومية لكن لا يمس المال أو الضرائب مباشرة | Regression قبل + Dual Run |
| **Tier 3** — ثانوي | أدوات إدارية أو تقارير — لا يؤثر على بيانات حية | اختبار بسيط |

## Dual Run Mode — كيف يعمل

### المبدأ
```
index.html (vanilla)     React App (جديد)
      ↓                        ↓
   /ms-accounting/          /ms-accounting/
   (الإنتاج الحالي)       (route مع /react prefix
                            أو feature flag)
      ↓                        ↓
      ↓←←←←← نفس الـ API ←←←←←↓
      ↓                        ↓
  نفس قاعدة البيانات ←←←←←←←←←↓
```

### آلية التنفيذ
- الـ React version ترد على نفس الـ API endpoints بالضبط
- المستخدمون يمكنهم استخدام النسختين في نفس الوقت
- أي write operation في React يظهر فوراً في vanilla (نفس الـ DB)
- الـ URL يحدد النسخة: `/ms-accounting/` = vanilla, `/ms-accounting/app/` = React

### ما يُقارَن خلال الـ Dual Run
| المعيار | كيف نتحقق |
|---------|----------|
| نفس البيانات | عدد السجلات في كلا الواجهتين |
| نفس النتائج | نفس الفلاتر تُعطي نفس النتائج |
| نفس الصلاحيات | Admin/Manager/Employee يرى نفس الشيء |
| نفس التقارير | الأرقام المالية متطابقة |
| Performance | وقت التحميل لا يزيد بأكثر من 50% |

## Rollback Plan — لكل Module

### ما هو الـ Rollback

إذا ظهرت مشكلة في الـ React version بعد بدء الـ Dual Run:

```
الوقت المستهدف للـ Rollback: أقل من 5 دقائق
```

### خطوات الـ Rollback

**الخطوة 1** — إيقاف الـ React version (إخفاء الرابط)
```javascript
// في config.js أو feature flags:
window.REACT_ENABLED = false;
// → كل المستخدمين يعودون للـ vanilla تلقائياً
```

**الخطوة 2** — التحقق من البيانات
```bash
# هل البيانات سليمة في DB؟
curl /api/admin/metrics/baseline
```

**الخطوة 3** — إذا تأثرت البيانات → استرجاع الـ Backup
```
في شاشة النسخ الاحتياطية → اختر آخر backup قبل النقل → Restore
```

**الخطوة 4** — تسجيل المشكلة
```
في KNOWN_ISSUES.md → سبب الـ Rollback + ما الذي أخطأ
```

**الخطوة 5** — إصلاح والمحاولة مجدداً
```
لا تعيد النقل حتى يُصلَح السبب الجذري
```

### مخطط قرار الـ Rollback

```
ظهرت مشكلة؟
    ↓
هل تمس البيانات (خسارة، تحريف، ازدواجية)؟
    ├── نعم → Rollback فوري + Restore backup
    └── لا  → هل تمنع المستخدمين من العمل؟
                ├── نعم → Rollback فوري (بدون Restore)
                └── لا  → هل يمكن إصلاحها خلال ساعة؟
                            ├── نعم → أصلح بدون Rollback
                            └── لا  → Rollback
```
